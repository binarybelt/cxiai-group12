    Create `src/lib/compliance.ts` implementing these exported functions (per research Patterns 1, 6, 7):

    1. `runBrandChecks(spec: PageSpec): ComplianceViolation[]`
       - Build approved component ID set from `loadComponents()`
       - Build approved token ID set from `loadTokens()`
       - For each section.components: check componentId against approved set (violation: "brand-component-only", severity: "error", autoFixable: false)
       - For each tokenOverrides entry: check tokenId against approved set (violation: "brand-approved-colors", severity: "error", autoFixable: true)

    2. `runPharmaChecks(spec: PageSpec): ComplianceViolation[]`
       - If market contains "hcp" (case-insensitive) and no component has componentId "ISIBlock": violation "pharma-isi-required-hcp", severity: "error"
       - For Footer components: if adverseEventUrl prop is missing/empty/"#": violation "pharma-adverse-event-link", severity: "error"
       - If no Disclaimer component and no "disclaimer" section type: violation "pharma-disclaimer-promotional", severity: "error"
       - For market requiredComponents from `getMarketConfig(spec.market)`: if missing, violation severity: "warning"

    3. `runComplianceGate(spec: PageSpec): { passed: boolean; violations: ComplianceViolation[] }`
       - Combine brand + pharma violations
       - passed = no "error" severity violations

    4. `computeScore(brand: ComplianceViolation[], pharma: ComplianceViolation[], a11y: { length: number }[]): ComplianceScore`
       - Start at 100, deduct 25 per error, 10 per warning. Floor at 0.
       - a11y score: 100 - (count * 20), floor at 0
       - overall = Math.round((brand + accessibility + pharma) / 3)

    5. `applyAutoFix(spec: PageSpec, violation: ComplianceViolation): PageSpec`
       - For "brand-approved-colors": filter out unapproved tokenOverrides. Return new object (spread, never mutate).
       - For non-auto-fixable violations: return spec unchanged.

    All functions are pure -- no side effects, no DOM, no React. Use immutable patterns per project coding style.

    Run tests: they must ALL pass (GREEN phase).
