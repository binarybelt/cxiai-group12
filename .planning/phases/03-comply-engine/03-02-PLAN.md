<objective>
Build the axe-core accessibility scanner wrapper and wire the compliance gate into the generate-page API route so that non-compliant PageSpecs are rejected with a 422 before any response reaches the client.

Purpose: COMPLY-03 requires WCAG AA accessibility scanning against rendered DOM. COMPLY-06 requires the pipeline to block rendering when the PageSpec fails compliance. This plan delivers both: the scanner as a browser-only module, and the gate as a server-side hard gate in the API route using generateObject (not streamObject) so the full spec is available for validation before responding.

  <action>
    Modify `src/app/api/generate-page/route.ts` to replace `streamObject` with `generateObject` so the full PageSpec is available before any response is sent. Then run `runComplianceGate` and return 422 on failure.

    1. Change import from `streamObject` to `generateObject`:
       ```typescript
       import { generateObject } from "ai";
       ```

    2. Add compliance imports:
       ```typescript
       import { runComplianceGate } from "@/lib/compliance";
       ```

    3. Replace the `streamObject` call and response handling with:
       ```typescript
       const result = await generateObject({
         model: getLLM(),
         schema,
         system: systemPrompt,
         prompt: JSON.stringify(interpretation),
         maxRetries: 2,
       });

       const spec = result.object;

       // Fire audit log
       void logGeneration(
         "generate-page",
         JSON.stringify({
           market: interpretation.market,
           product: interpretation.product,
           pageType: interpretation.pageType,
           variantCount: spec?.variants?.length ?? 0,
           tokenIds: tokenIds.slice(0, 5),
         }),
       );

       // COMPLY-06: Compliance gate — block non-compliant specs
       // Run gate against each variant; fail if ANY variant has errors
       const allViolations = (spec.variants ?? []).flatMap((variant: PageSpec) => {
         const gateResult = runComplianceGate(variant);
         return gateResult.violations;
       });
       const hasErrors = allViolations.some(
         (v: ComplianceViolation) => v.severity === "error"
       );

       if (hasErrors) {
         return Response.json(
           {
             error: "Compliance gate blocked rendering",
             violations: allViolations,
             spec, // include spec so client can show violations in sidebar
           },
           { status: 422 },
         );
       }

       // Happy path: return the generated spec as JSON
       return Response.json(spec);
       ```

    4. Import ComplianceViolation type for the severity check:
       ```typescript
       import type { ComplianceViolation } from "@/types/compliance";
       ```

    IMPORTANT: This changes the response format from a text stream to JSON. Plan 03-03 Task 2 will update BuildUI to use fetch instead of useObject to match this new response format.

    Do NOT add onFinish callback — generateObject completes synchronously (awaited), so audit logging and gate checking happen inline.
