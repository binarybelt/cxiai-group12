1. Install Convex: `npm install convex`

2. Run `npx convex dev --once` to initialize the Convex project. This creates the `convex/` directory with `_generated/` files and sets up `.env.local` with `CONVEX_DEPLOYMENT` and `NEXT_PUBLIC_CONVEX_URL`.

   If prompted for project name, use "design-delivery-accelerator". If prompted for team, select the default.

3. Create `convex/schema.ts`:
   - Define an `auditLog` table with fields:
     - `action` (string) — what happened (e.g., "page_generated", "edit_applied", "compliance_check")
     - `details` (string) — JSON string with structured details
     - `timestamp` (number) — Date.now() epoch ms
     - `actor` (string) — who/what triggered it (e.g., "user", "brief-interpreter", "compliance-gate")
   - Index on `by_timestamp` (timestamp field) for chronological queries

4. Create `convex/auditLog.ts`:
   - Export a `logAction` mutation that accepts { action, details, actor } and inserts into auditLog with timestamp = Date.now()
   - Export a `getRecentLogs` query that returns the last 50 entries ordered by timestamp desc

5. Create `src/lib/convex.ts`:
   - Export a configured ConvexReactClient using `process.env.NEXT_PUBLIC_CONVEX_URL!`

6. Update `src/app/layout.tsx`:
   - Wrap children with `ConvexProvider` from `convex/react` using the client from `src/lib/convex.ts`
   - Mark as "use client" or create a separate `providers.tsx` client component and keep layout as server component

7. Run `npx convex dev --once` again to push the schema.

8. Verify by checking `npx convex dashboard` opens or by confirming schema push succeeded.
