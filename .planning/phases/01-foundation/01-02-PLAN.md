  <files>src/types/page-spec.ts, src/types/design-system.ts, src/types/page-spec.test.ts</files>
  <behavior>
    - A valid PageSpec with sections, component refs, and metadata passes validation
    - A PageSpec missing required fields (e.g., no sections) fails validation
    - ComponentRef requires componentId (string), props (Record), and optional tokenOverrides
    - Section requires id, type (hero|content|cta|footer|navigation|data|disclaimer), and components array
    - PageSpec requires id, title, market, product, sections array, and metadata
    - Design system types (DesignToken, ComponentDef, Pattern, ComplianceRule, MarketConfig) are exportable and usable
  </behavior>
  <action>
Install Zod: `npm install zod`

1. Create `src/types/design-system.ts`:
   - `DesignToken` type: { id, category (color|typography|spacing|radius|shadow), name, value, description? }
   - `ComponentDef` type: { id, name, category, description, props: PropDef[], variants: string[], constraints: string[], requiredTokens: string[] }
   - `PropDef` type: { name, type (string|boolean|enum|node), required, default?, options?: string[] }
   - `Pattern` type: { id, name, description, sections: string[], requiredComponents: string[] }
   - `ComplianceRule` type: { id, category (brand|accessibility|pharma), severity (error|warning|info), description, check: string, autoFixable: boolean }
   - `MarketConfig` type: { id, name, region, requiredComponents: string[], requiredDisclosures: string[], additionalRules: string[] }
   - `DesignSystem` type: { tokens, components, patterns, complianceRules, markets } — the aggregate

2. Create `src/types/page-spec.ts`:
   - Use Zod schemas (not just types) — these are used for RUNTIME validation
   - `TokenOverrideSchema`: z.object({ tokenId: z.string(), value: z.string() })
   - `ComponentRefSchema`: z.object({ componentId: z.string(), props: z.record(z.unknown()), tokenOverrides: z.array(TokenOverrideSchema).optional() })
   - `SectionSchema`: z.object({ id: z.string(), type: z.enum(["hero", "content", "cta", "footer", "navigation", "data", "disclaimer"]), components: z.array(ComponentRefSchema), order: z.number() })
   - `PageSpecMetadataSchema`: z.object({ generatedBy: z.string(), generatedAt: z.string(), complianceScore: z.number().min(0).max(100).optional(), market: z.string(), product: z.string() })
   - `PageSpecSchema`: z.object({ id: z.string(), title: z.string(), market: z.string(), product: z.string(), sections: z.array(SectionSchema).min(1), metadata: PageSpecMetadataSchema })
   - Export inferred types: `type PageSpec = z.infer<typeof PageSpecSchema>` etc.

3. Write tests first in `src/types/page-spec.test.ts` per the behavior spec above, then implement.
